# Instructor Guide

## Course Overview

This guide provides instructors with essential information for teaching the Physical AI & Humanoid Robotics course effectively.

## Prerequisites

### Technical Requirements
- **Workstation**: Minimum 16GB RAM, 8+ core processor, dedicated GPU (NVIDIA recommended)
- **Operating System**: Ubuntu 22.04 LTS or Windows 10/11 with WSL2
- **Software Stack**:
  - ROS 2 Humble Hawksbill
  - Gazebo Garden
  - NVIDIA Isaac Sim (optional for advanced modules)
  - Python 3.10+
  - Git version control

### Hardware Requirements
- NVIDIA Jetson development kit (optional but recommended)
- RGB-D camera (Intel Realsense or equivalent)
- Basic sensors (IMU, encoders)

## Course Structure

### Module 1: Robotic Nervous System (ROS 2)
- Duration: 3-4 weeks
- Focus: ROS architecture, nodes, topics, services
- Assessment: Basic ROS node implementation

### Module 2: Digital Twin (Gazebo & Unity)
- Duration: 3-4 weeks
- Focus: Simulation environments, physics modeling
- Assessment: Robot simulation in virtual environment

### Module 3: AI-Robot Brain (NVIDIA Isaac)
- Duration: 3-4 weeks
- Focus: Perception, navigation, manipulation
- Assessment: Autonomous navigation task

### Module 4: Vision-Language-Action (VLA)
- Duration: 3-4 weeks
- Focus: Human-robot interaction, task planning
- Assessment: Voice-commanded robot operation

## Teaching Strategies

### Active Learning
- Encourage hands-on experimentation
- Provide real-time feedback during lab sessions
- Use pair programming for complex implementations

### Assessment Methods
- Weekly practical assignments
- Module-specific projects
- Capstone project demonstrating integrated knowledge
- Peer review sessions for code quality

## Common Student Challenges

### ROS Concepts
- Understanding the publisher-subscriber pattern
- Managing distributed systems
- Debugging communication issues

### Simulation Physics
- Understanding realistic physics parameters
- Collision detection and response
- Sensor noise modeling
